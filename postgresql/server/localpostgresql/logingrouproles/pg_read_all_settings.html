pg_read_all_settings#

What is pg_read_all_settings?#

pg_read_all_settings is a built-in role that allows a user to read all PostgreSQL configuration settings.

In simple words:

‚ÄúI can see how the server is configured.‚Äù

What pg_read_all_settings CAN do#

A role with this privilege can:

Read all configuration parameters

Including:

  • shared_buffers

  • work_mem

  • maintenance_work_mem

  • max_connections

  • wal_level

  • log_statement

  • ssl, ssl_ciphers

  • archive_mode

  • autovacuum_*

  • listen_addresses

From:

SHOW ALL;

Or:

SELECT * FROM pg_settings;
What it CANNOT do

  • ‚ùå Change settings

  • ‚ùå Reload config

  • ‚ùå Restart server

  • ‚ùå Read data from tables

  • ‚ùå Access files

  • ‚ùå Execute OS commands

It is read-only.

Why this role exists

Normally:

SELECT * FROM pg_settings;

returns only non-sensitive settings

But with pg_read_all_settings:

  • You see everything

  • Including hidden / sensitive parameters
